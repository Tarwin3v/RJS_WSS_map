### DEPENDENCIES

## CLIENT

# apollo-cache-inmemory :: cache implementation for Apollo Client 2.0

- https://www.npmjs.com/package/apollo-cache-inmemory

# apollo-client :: Apollo Client is a fully-featured caching GraphQL client

- https://www.npmjs.com/package/apollo-client

# apollo-link-ws :: This link is particularly useful to use GraphQL Subscriptions, but it will also allow you to send GraphQL queries and mutations over WebSockets as well.

- https://www.npmjs.com/package/apollo-link-ws

# date-fns :: toolset for manipulating JavaScript dates in a browser & Node.js

- https://www.npmjs.com/package/date-fns

# graphql-request :: Minimal GraphQL client supporting Node and browsers for scripts or simple apps

- https://www.npmjs.com/package/graphql-request

# graphql-tag :: lets you write your queries with ES2015 template literals and compile them into an AST with the gql tag

- https://www.npmjs.com/package/graphql-tag

# mapbox-gl :: Mapbox GL JS is a JavaScript library that uses WebGL to render interactive maps

- https://docs.mapbox.com/mapbox-gl-js/api/

# react-apollo :: React Apollo allows you to fetch data from your GraphQL server

- https://www.apollographql.com/docs/react/api/react-apollo/

# react-google-login :: A Google oAUth Sign-in / Log-in Component for React

- https://www.npmjs.com/package/react-google-login

# react-map-gl :: eact-map-gl is a suite of React components for Mapbox GL JS

- https://www.npmjs.com/package/react-map-gl

# subscriptions-transport-ws :: A GraphQL WebSocket server and client to facilitate GraphQL queries, mutations and subscriptions over WebSocket

- https://www.npmjs.com/package/subscriptions-transport-ws

## SERVER

# apollo-server :: Apollo Server is a community-maintained open-source GraphQL server. It works with pretty much all Node.js HTTP server frameworks

- https://www.npmjs.com/package/apollo-server

# google-auth-library :: This is Google's officially supported node.js client library for using OAuth 2.0

- https://www.npmjs.com/package/google-auth-library
